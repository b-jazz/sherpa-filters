No To: Header Line

It continues to baffle me why spammers don't follow the RFC rules on how messages should be formatted. One of the things that spam generating software does is fail to include a To: header line. Most, if not all, legitimate mailers will include a To: line even if there aren't any addresses in the To: line.

The example filter below will see if the header doesn't even exist and will filter the message away as spam.

Example

<Filter Name="SPAM: No To: Header Line" Enabled="true" Tag="Spam">
  <Criteria>
    <Not>
      <HeaderExists>
        <Header>To</Header>
      </HeaderExists>
    </Not>
  </Criteria>
  <Actions>
    <Move>
      <Destination>Mailboxes/Spam</Destination>
    </Move>
  </Actions>
  <Stats>
    <TestCounter>0</TestCounter>
    <MatchCounter>0</MatchCounter>
    <FilterTime>0</FilterTime>
    <LastMatchTime></LastMatchTime>
  </Stats>
</Filter>
