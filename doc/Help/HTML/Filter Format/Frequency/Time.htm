Time

Specify the minimum amount of time before acting on a stream of duplicate email messages.

Just like the Count option in the Frequency setting, the Time option will instead wait the specified amount of time before acting on a matching email message. This way, you can opt to only receive one email message per hour from a failing script.

If you haven't seen your script failure email for a few weeks, and then you get twenty of them in a row, the first one will be considered a match, and then the next several won't trigger a filter match for the specified amount of time. After that amount of time goes by, the next message that matches the criteria will trigger a filter match and will perform the filter's actions.

It's a good idea to have a pair of filters when using the Frequency ability. The first filter will grab the occassional match, and the second filter will toss the bulk of the repeat messages.

The format of the value you specify is [DDD.]HH:MM:SS. The number of days is optional, but the hours, minutes and seconds must be specified and must be exactly two digits.

Time Example

This is the same example as the Count option, but accepts only one message every 59 minutes and 5 seconds.

<Filter Name="Count Example">
  <Criteria>
    <HeaderMatch>
      <Header>Subject</Header>
      <Value>^Script Failure: </Value>
    </HeaderMatch>
  </Criteria>
  <Frequency>
    <Time>00:59:05</Time>
  </Frequency>
  <Actions>
    ...
  </Actions>
  <Stats>
    ...
  </Stats>
</Filter>
