RelayDomain

Checks to see if any of the email servers that handled the email belongs to (or doesn't belong to) one of the listed top-level domains.

You can either specify a list of Good domains (a sort of whitelist), or a list of Bad domains (a blacklist). The list of values are not regular expressions and should include the dot (.) for readability.

RelayDomain Example

In the example below, we list the major top level domains that send out mail. Not a whole lot of spam originates in the UK, so we allow them to, and since we have an Au Pair from Poland that sends us mail, we include .pl in the list of good domains.

<Criteria>
  <RelayDomain>
    <Good>.com</Good>
    <Good>.net</Good>
    <Good>.edu</Good>
    <Good>.gov</Good>
    <Good>.org</Good>
    <Good>.us</Good>
    <Good>.uk</Good>
    <Good>.pl</Good>
  </RelayDomain>
</Criteria>

If you don't want to be as strict, you could use the example below to list out domains that you know do send a lot of spam. I've seen a fair amount of spam from North Korea, Nigeria, and Russia. I'd start there and add more top level domains as you see an increase in spam from them.

<Criteria>
  <RelayDomain>
    <Bad>.kr</Bad>
    <Bad>.ng</Bad>
    <Bad>.ru</Bad>
  </RelayDomain>
</Criteria>
