Filter Attributes

Each filter has some attributes that are specified within the Filter tag. Some of them are mandatory, others are optional.

Attribute Required Default Description
Name Yes none The name you give to the filter. This is used when logging information to the event log and will be used later in the user interface.
Enabled No "true" Whether or not the filter is currently enabled. If the filter isn't enabled, it will be skipped when messages are fed through the available filters. Possible values are "true", "false", 1, and 0.
Tag No "Normal" Defines how the matching email message should be tagged, whether it counts as a spam message, a normal message, or if the message doesn't affect the counters. This is mainly used for the statistics screen showing you how much spam you've received. In the future, it might be used in conjunction with the Bayes filter and improving the collections of spam and normal mail. Possible values are "Normal", "Spam", and "Ignore".

Filter Attribute Examples

This example shows a filter that is used to catch email addresses from people that I know. The tag isn't specified and defaults to normal email and will adjust the "normal" counters whenever the filter matches.

<Filter Name="IN: Email From Known Contacts" Enabled="true">
  <Criteria> ... </Criteria>
  <Actions>
    <Move>
      <Destination>Mailboxes/In</Destination>
    </Move>
  </Actions>
  <Stats> ... </Stats>
</Filter>

This is an example of a spam filter. It is enabled by default, but the Tag is set to "Spam" to go against the Spam counters.

<Filter Name="SPAM: Email From Known Spammers" Tag="Spam">
  <Criteria> ... </Criteria>
  <Actions>
    <Move>
      <Destination>Mailboxes/Spam</Destination>
    </Move>
  </Actions>
  <Stats> ... </Stats>
</Filter>
